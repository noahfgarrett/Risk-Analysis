# Repository Instructions

- Keep changelog and GitHub release notes intentionally vague and user-facing.
- Prefer short phrases such as "Minor dashboard refinements" or "Small update flow improvements" over detailed implementation notes.
- Do not mention internal helper names, test names, or exact technical mechanics in update notes unless the user explicitly asks for detailed release notes.
- Attach only the plain `Risk-Analysis.html` file to releases unless the user explicitly asks for additional release assets.
